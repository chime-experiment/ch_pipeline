class ch_pipeline.analysis.beam.MakeHolographyBeam[source]¶

Bases: draco.core.task.SingleTask

Repackage a holography transit into a beam container.

The visibilities will be grouped according to their respective 26 m input (along the pol axis, labelled by the 26 m polarisation of that input).

The form of the beam dataset is A_{i} = < E_{i} * E_{26m}^{*} >, i.e. the 26m input is the conjugate part of the visbility.

Initialize pipeline task.

May be overridden with no arguments. Will be called after any config.Property attributes are set and after ‘input’ and ‘requires’ keys are set up.

class ch_pipeline.analysis.beam.TransitStacker[source]¶

Bases: draco.core.task.SingleTask

Stack a number of transits together.

The weights will be inverted and stacked as variances. The variance between transits is evaluated and recorded in the output container.

All transits should be on a common grid in hour angle.
